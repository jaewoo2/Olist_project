# Olist 2016-2018 데이터 분석
## 고객 재정의를 통한 전략적 인사이트 도출

**분석 기간:** 2016년 9월 - 2018년 8월  
**데이터 출처:** Olist Public Dataset (Kaggle)  
**분석 목적:** 포트폴리오 프로젝트 (8주, 7인 팀)

---

## 📌 Executive Summary (면접관용)

### 핵심 질문
**"데이터 분석가가 비즈니스를 얼마나 이해하는가?"**

### 이 프로젝트가 증명하는 것
1. **데이터 리터러시:** 100k 주문 데이터에서 핵심 문제(배송-만족도-재구매) 발견
2. **비즈니스 센스:** 피터 드러커 "고객은 누구인가" 질문으로 B2B 본질 파악
3. **전략적 사고:** 데이터 → 인사이트 → 실행 가능한 전략 제안
4. **검증 가능성:** 이후 Olist의 실제 행동(2020-2021)으로 접근법 검증됨

### 차별화 포인트
- ❌ "배송이 문제다" (피상적)
- ✅ "판매자가 진짜 고객이며, 그들의 배송 품질을 높여야 한다" (본질적)

## 1. 분석 방향성 및 접근법

### 1.1 프로젝트 배경

#### Olist란?
```
비즈니스 모델: B2B E-commerce Enabler
설립: 2015년
역할: 중소 판매자를 대형 플랫폼(Mercado Livre, Amazon 등)에 연결

특징:
- 판매자가 Olist 플랫폼 사용 → 여러 마켓플레이스 통합 관리
- 구매자는 Mercado Livre 등에서 구매 → Olist 브랜드 인지도 낮음
- 배송: 판매자 직접 수행 (당시 Olist 통제 불가)
```

#### 분석 시점 맥락 (2016-2018)
```
Olist 규모:
- 연 GMV: $7-8.6M
- 판매자: 수천 명 (추정)
- 위치: 초기 단계 스타트업

브라질 시장:
- 전체 이커머스: $20-30B
- Mercado Libre: 25-30%
- B2W: 20-25%
- Olist GMV 점유율: 0.03%
```

### 1.2 분석 접근법

#### 전통적 접근의 한계
```
❌ 일반적 분석:
"고객(구매자) 만족도를 높이자"
→ 배송 개선 → 하지만 Olist가 통제 못 함 → 막힘

문제:
- 구매자 = Olist의 직접 고객 아님
- Olist는 중개자일 뿐
- 실행 가능성 없는 제안
```

#### 우리의 접근: 드러커 질문 활용
```
✅ 피터 드러커:
"우리 고객은 누구인가?"

이 질문이 중요한 이유:
1. 비즈니스 본질 파악
2. 제한된 리소스의 집중
3. 실행 가능한 전략 도출

방법론:
1. 데이터로 문제 발견 (구매자 관점)
2. 드러커로 고객 재정의 (판매자 관점)
3. 통합 전략 제안 (양면 플랫폼 관점)
```

### 1.3 데이터셋 개요

#### 구조
```
기간: 2016년 9월 - 2018년 8월 (약 24개월)
주문 수: 99,441건
고객 수: 96,096명
판매자 수: 3,095명
```

#### 주요 테이블
```
1. orders: 주문 기본 정보
2. order_items: 주문 상품 상세
3. order_payments: 결제 정보
4. order_reviews: 리뷰 점수 및 텍스트
5. customers: 고객 위치 정보
6. sellers: 판매자 위치 정보
7. products: 상품 카테고리
8. geolocation: 지리 좌표
```

#### 핵심 변수
```
배송:
- order_purchase_timestamp
- order_delivered_customer_date
- order_estimated_delivery_date
→ 배송 지연 계산 가능

만족도:
- review_score (1-5점)
- review_comment_message

재구매:
- customer_unique_id로 추적 가능
```

## 2. 고객 재정의 (피터 드러커)

### 2.1 드러커의 5가지 질문

#### Q1. 우리 사업은 무엇인가?
```
표면적 답변:
"이커머스 플랫폼"

본질적 답변:
"중소 판매자의 온라인 진출 지원"

차이:
- 전자 → 구매자에게 물건 파는 회사
- 후자 → 판매자가 물건 팔도록 돕는 회사
```

#### Q2. 우리 고객은 누구인가?
```
착각 1: "구매자"
- 구매자는 Mercado Livre 고객
- Olist 브랜드 인지도 낮음
- 재구매율 극히 낮음

착각 2: "판매자"
- 맞지만 부분적
- 판매자가 수수료 지불 = 고객 맞음

정답: "판매자가 Primary, 구매자가 Secondary"
- 판매자 = 직접 고객 (서비스 선택)
- 구매자 = 간접 고객 (판매자 통해 영향)
```

#### Q3. 고객은 무엇을 가치 있게 여기는가?
```
판매자:
✅ 대형 플랫폼 접근 (Mercado Livre 진입)
✅ 통합 관리 시스템 (하나의 계약으로 여러 채널)
✅ 매출 증가

구매자:
✅ 상품 다양성
❌ 배송 품질 (문제!)
❌ 일관된 경험 (부족!)
```

#### Q4. 우리의 결과는 무엇인가?
```
기존 지표 (잘못됨):
- 구매자 만족도
- 재구매율
→ 이건 판매자 책임

올바른 지표:
- 판매자 성공률 (매출 증가 %)
- 판매자 리텐션 (이탈률)
- 판매자당 평균 GMV
```

#### Q5. 우리의 계획은 무엇인가?
```
기존 사고:
"구매자 만족 → 재구매 증가"
→ 하지만 배송 통제 못 함

드러커 사고:
"판매자 성공 → 구매자 만족 → 판매자 더 성공"
→ 판매자 배송 품질 향상 지원
```

### 2.2 데이터 기반 검증

#### 구매자가 주 고객이 아닌 증거
```
데이터 근거:

1. 재구매율 분석 (2017년 11월 신규 고객 코호트)
   - 1개월 후 재구매: 0.6%
   - 전체 재구매율: 극히 낮음
   → 구매자는 Olist를 "선택"하지 않음

2. 브랜드 인지도
   - 구매자는 "Mercado Livre에서 샀다"고 인식
   - Olist 브랜드 노출 최소
   → Olist는 구매자에게 "투명"함

3. 가치 전달 경로
   - Olist → 판매자 → 구매자
   - 직접 가치 전달: 판매자에게만
```

#### 판매자가 주 고객인 증거
```
데이터 근거:

1. 판매자 선택
   - 판매자가 Olist 플랫폼 "선택"
   - 수수료 지불
   - 서비스 지속 사용 결정

2. 매출 구조
   - Olist 수익 = 판매자 수수료
   - 판매자 이탈 = Olist 매출 감소
   
3. 성장 동력
   - 판매자 수: 2016년 수십 명 → 2018년 3,095명
   - GMV: $7M (2017) → $8.6M (2018)
   - 판매자 성장 = Olist 성장
```

### 2.3 전략적 시사점

#### 관점 전환
```
Before (구매자 중심):
문제: "배송 불만 → 재구매 낮음"
해결: "배송 개선" → 하지만 불가능 (통제 못 함)

After (판매자 중심):
문제: "판매자 배송 품질 → 구매자 불만 → 판매자 매출 영향"
해결: "판매자 배송 품질 향상 지원" → 가능!
```

#### 실행 가능성
```
구매자 중심 전략:
❌ Olist가 직접 배송 (자본 필요, 당시 불가)
❌ 배송 속도 보장 (통제 불가)
❌ 구매자 리워드 (재방문 보장 안 됨)

판매자 중심 전략:
✅ 배송 우수 판매자 발굴 및 벤치마킹
✅ 판매자 교육 (배송 중요성)
✅ 판매자 등급제 (배송 품질 포함)
✅ 우수 판매자 인센티브
```

## 3. 데이터 기반 문제 발견

### 3.1 탐색적 데이터 분석 (EDA)

#### 3.1.1 배송 시간 분석
```
배송 시간 분포:
- 중앙값: 8일
- 평균: 12.5일
- 표준편차: 10.2일
→ 편차 큼 = 일관성 없음

지연 건수:
- 예상일 내 도착: 92.7%
- 지연: 7.3% (약 7,200건)

치명적 지연:
- 10일 이상 지연: 2.1%
- 20일 이상 지연: 0.8%
```

#### 3.1.2 배송 시간과 리뷰 점수 관계
```
상관관계 분석:
- 배송 시간 vs 리뷰 점수: r = -0.36 (중간 음의 상관)
- 통계적 유의성: p < 0.001

세그먼트 분석:
배송 시간    평균 리뷰 점수    5점 비율
0-7일       4.2점            65%
8-15일      4.0점            55%
16-30일     3.5점            35%
30일+       2.8점            18%

→ 배송 오래 걸릴수록 만족도 급락
```

#### 3.1.3 리뷰 점수와 재구매 관계
```
재구매 분석 (2017년 11월 신규 고객 코호트 기준):

리뷰 점수    재구매율 (1개월)
5점         1.2%
4점         0.8%
3점         0.4%
1-2점       0.1%

전체 평균: 0.6%

핵심 발견:
- 5점 줘도 재구매율 1.2%
- 극히 낮은 로열티
- 브랜드 인지도/충성도 없음
```

### 3.2 핵심 문제 정의

#### 문제 1: 배송 품질의 일관성 부족
```
데이터 근거:
- 표준편차 10.2일 (평균 12.5일)
- 같은 주(state) 내에서도 편차 큼
- 판매자별 편차: 최소 3일 ~ 최대 45일

원인:
- 판매자가 직접 배송
- 판매자마다 다른 택배사/물류
- Olist의 품질 통제 없음

영향:
- 구매자 불만
- 낮은 리뷰 점수
- 판매자 매출 영향
```

#### 문제 2: 구조적 한계
```
Olist 비즈니스 모델 (2016-2018):
├─ 판매자 → (직접 배송) → 구매자
└─ Olist: 중개만, 배송 개입 없음

한계:
❌ 배송 속도 보장 불가
❌ 배송 품질 통제 불가
❌ 일관된 고객 경험 제공 불가

결과:
→ 구매자 불만 → 판매자에게 전가
→ 우수한 판매자도 플랫폼 이탈 위험
```

#### 문제 3: 재구매 메커니즘 부재
```
데이터 근거:
- 재구매율: 0.6% (극히 낮음)
- 99.4% 고객이 1회 구매 후 이탈
- 5점 리뷰 줘도 1.2%만 재구매

원인:
1. 브랜드 인지도 없음
   → 구매자는 "Mercado Livre에서 샀다"고 인식
   
2. 플랫폼 충성도 없음
   → Olist 경험이 아닌 개별 판매자 경험
   
3. 구조적 문제
   → 다음 구매 시 다른 판매자 = 다른 경험
   → 일관성 없음

영향:
→ 고객 획득 비용(CAC) 회수 불가
→ LTV 낮음
→ 성장 한계
```

### 3.3 문제의 본질

#### 표면적 문제
```
"배송이 느리다"
"고객 만족도가 낮다"
"재구매율이 낮다"
```

#### 본질적 문제
```
"Olist는 B2B 플랫폼이지만
 B2C 고객 경험을 책임져야 하는 모순"

구조:
판매자(고객) → 배송 직접 → 구매자 불만
    ↑                           ↓
 Olist 수익 ←─────────────── 매출 감소

→ Olist의 성공 = 판매자의 성공
→ 판매자의 성공 = 구매자 만족
→ 하지만 Olist는 구매자 만족에 직접 개입 불가

해결 방향:
판매자가 배송 품질을 높이도록 "돕는" 것
```

## 4. 가설 및 검증 계획

### 4.1 가설 설정 원칙

#### 검증 가능성 (Testability)
```
✅ 좋은 가설:
"배송 시간이 10일을 넘으면 리뷰 점수가 유의미하게 하락한다"
→ 데이터로 검증 가능

❌ 나쁜 가설:
"배송 개선하면 매출이 증가할 것이다"
→ 인과관계 불명확, 측정 어려움
```

#### 실행 가능성 (Actionability)
```
✅ 좋은 가설:
"배송 우수 판매자의 특징을 파악하여 다른 판매자에게 교육할 수 있다"
→ 실행 가능

❌ 나쁜 가설:
"브라질 물류 인프라를 개선해야 한다"
→ Olist 통제 불가
```

### 4.2 구매자 관점 가설 (문제 발견용)

#### H1: 지역별 배송 격차
```
가설:
"동남부와 북동부 간 배송 시간 차이가 2배 이상이며,
 이것이 지역별 만족도 차이로 이어진다"

근거:
- 브라질 지역 간 경제 격차 (소득 4배 차이)
- 물류 인프라 차이
- 인구 밀집도 차이

검증 방법:
1. 주(state)별 평균 배송 시간 계산
2. 동남부 vs 북동부 평균 비교
3. 지역별 리뷰 점수 분포 비교
4. 통계적 유의성 검정 (t-test)

기대 결과:
- 동남부: 7-10일
- 북동부: 14-20일
- 리뷰 점수: 동남부 > 북동부

비즈니스 활용:
→ 지역별 배송 전략 차별화
→ 북동부 판매자 특별 지원
```

#### H2: 배송 지연 임계점
```
가설:
"배송 지연이 특정 일수(예: 10일)를 넘어서면
 리뷰 점수가 급격히 하락한다"

근거:
- 고객의 심리적 인내 한계
- 예상 배송일 대비 실제 지연

검증 방법:
1. 배송 지연 = 실제 배송일 - 예상 배송일
2. 지연 일수별 리뷰 점수 분포
3. 변곡점(elbow point) 탐색
4. 세그먼트별 회귀 분석

기대 결과:
- 0-5일 지연: 리뷰 4.0+
- 6-10일 지연: 리뷰 3.5
- 10일+ 지연: 리뷰 3.0 이하 급락

비즈니스 활용:
→ 임계점 이내 배송 판매자 우선 노출
→ 지연 위험 판매자 조기 경고 시스템
```

#### H3: 카테고리별 배송 민감도
```
가설:
"가구/가전 같은 고가 카테고리는
 배송 지연에 더 민감하게 반응한다"

근거:
- 고객의 기대치 (비싼 물건 = 빠른 배송)
- 제품 가치 (배송 중 파손 우려)

검증 방법:
1. 카테고리별 분류
2. 카테고리 × 배송 시간 → 리뷰 점수 교차 분석
3. 카테고리별 회귀 계수 비교

기대 결과:
- 고가 카테고리: 배송 시간 계수 크게 음수
- 저가 카테고리: 배송 시간 계수 작게 음수

비즈니스 활용:
→ 카테고리별 배송 SLA 차별화
→ 고가 카테고리 판매자 특별 관리
```

#### H4: 결제 방법과 만족도
```
가설:
"신용카드 결제 고객이 볼레토(현금) 고객보다
 배송 지연에 더 민감하다"

근거:
- 신용카드 = 고소득 = 높은 기대
- 볼레토 = 저소득 = 낮은 기대

검증 방법:
1. 결제 방법별 고객 분류
2. 배송 시간 vs 리뷰 점수 (결제 방법별)
3. 회귀 분석 (교호작용 항 포함)

비즈니스 활용:
→ 결제 방법 기반 배송 전략
→ 신용카드 고객 우선 배송
```

#### H5: 첫 경험의 중요성
```
가설:
"첫 구매 경험(배송+상품)이 좋으면
 재구매 확률이 유의미하게 증가한다"

근거:
- 첫인상 효과 (First Impression)
- 신뢰 형성의 시작점

검증 방법:
1. 신규 고객 식별
2. 첫 구매 리뷰 점수 vs 재구매 여부
3. 로지스틱 회귀 분석

기대 결과:
- 첫 구매 5점 → 재구매율 2-3%
- 첫 구매 1-3점 → 재구매율 0.1%

비즈니스 활용:
→ 신규 고객 첫 주문 특별 관리
→ 신규 고객에게는 우수 판매자만 노출
```

### 4.3 판매자 관점 가설 (해결책 도출용)

#### H6: 판매자 배송 품질 차이
```
가설:
"판매자 간 배송 품질(속도/일관성) 차이가 크며,
 배송 우수 판매자는 매출/리뷰에서 우위를 점한다"

근거:
- 판매자가 직접 배송 → 역량 차이
- 우수한 판매자 = 배송 시스템 확립

검증 방법:
1. 판매자별 평균 배송 시간 계산
2. 판매자별 배송 시간 표준편차 (일관성)
3. 배송 품질 vs 판매자 매출 상관관계
4. 배송 품질 vs 평균 리뷰 점수 상관관계

기대 결과:
- 상위 10% 판매자: 평균 5일, 표준편차 1일
- 하위 10% 판매자: 평균 18일, 표준편차 8일
- 배송 품질 → 매출 강한 양의 상관

비즈니스 활용:
→ 배송 우수 판매자 TOP 10% 벤치마킹
→ 베스트 프랙티스 문서화
→ 전체 판매자 교육 자료로 활용
```

#### H7: 지리적 이점
```
가설:
"대도시(상파울루, 리우) 근처 판매자가
 배송 시간에서 유리하며 더 높은 매출을 올린다"

근거:
- 물류 허브 접근성
- 인구 밀집 = 수요 집중

검증 방법:
1. 판매자 위치 분류 (대도시 vs 기타)
2. 그룹 간 배송 시간 비교
3. 그룹 간 매출 비교

비즈니스 활용:
→ 지역별 판매자 모집 전략
→ 지방 판매자 물류 지원 필요성 파악
```

#### H8: 판매자 학습 곡선
```
가설:
"플랫폼 가입 후 시간이 지날수록
 판매자의 배송 품질이 향상된다"

근거:
- 경험 축적
- 피드백 학습
- 시스템 최적화

검증 방법:
1. 판매자별 가입 시점 파악
2. 시간 경과별 배송 시간 추이
3. 패널 회귀 분석

비즈니스 활용:
→ 신규 판매자 온보딩 프로그램
→ 초기 집중 지원
```

#### H9: 판매자 이탈 예측
```
가설:
"배송 품질이 나쁜 판매자는
 리뷰 점수 하락 → 매출 감소 → 이탈 위험 높음"

근거:
- 부정적 피드백 루프
- 수익성 악화

검증 방법:
1. 판매자별 활동 추적
2. 이탈 판매자 식별
3. 이탈 판매자 특징 (배송, 리뷰, 매출)
4. 예측 모델 (로지스틱 회귀)

비즈니스 활용:
→ 이탈 위험 판매자 조기 발견
→ 선제적 개입 (교육, 지원)
```

## 5. 전략 제안

### 5.1 전략 방향

#### 핵심 원칙
```
1. 판매자 = Primary Customer
   → 판매자 성공에 집중

2. 구매자 만족 = 판매자 성공의 결과
   → 간접 접근

3. 실행 가능성 우선
   → Olist가 통제 가능한 것부터
```

#### 3단계 전략
```
Phase 1 (즉시 실행 - 3개월):
→ 데이터 기반 판매자 지원

Phase 2 (단기 - 6-12개월):
→ 판매자 등급제 & 인센티브

Phase 3 (중장기 - 1-2년):
→ 물류 인프라 투자 검토
```

### 5.2 Phase 1: 데이터 기반 판매자 지원

#### 전략 1-1: 배송 우수 판매자 벤치마킹
```
목표:
배송 TOP 10% 판매자의 특징을 파악하여
다른 판매자가 학습할 수 있게 함

실행:
1. 배송 우수 판매자 선정 기준
   - 평균 배송 시간 7일 이내
   - 배송 시간 표준편차 2일 이내
   - 지연율 3% 이하

2. 심층 분석
   - 어떤 택배사 사용?
   - 어떻게 재고 관리?
   - 어떻게 포장/출고?

3. 베스트 프랙티스 문서화
   - "배송 7일 달성 가이드"
   - 체크리스트, 템플릿 제공

4. 전체 판매자 교육
   - 온라인 교육 콘텐츠
   - 웨비나
   - 1:1 컨설팅 (대형 판매자)

투자:
- 인력: 3명 (데이터 분석가 1명, PM 1명, 교육 담당 1명)
- 시간: 2-3개월
- 비용: 낮음 (인건비만)

기대 효과:
- 전체 판매자 평균 배송 시간 10% 개선
- 하위 50% 판매자 배송 시간 20% 개선
- 판매자 만족도 증가
```

#### 전략 1-2: 배송 실시간 모니터링 대시보드
```
목표:
판매자가 자신의 배송 퍼포먼스를 실시간 확인

실행:
1. 판매자 대시보드 구축
   표시 지표:
   - 평균 배송 시간
   - 지연율
   - 플랫폼 평균 대비 위치
   - 리뷰 점수 추이

2. 알림 시스템
   - 지연 위험 주문 경고
   - 리뷰 점수 하락 알림

3. 개선 제안
   - "이번 주 배송 시간이 평균보다 3일 깁니다"
   - "TOP 10% 판매자는 이렇게 합니다"

투자:
- 기술: 대시보드 개발
- 시간: 2개월
- 비용: 중간

기대 효과:
- 판매자 자가 개선
- 문제 조기 발견
- 플랫폼 투명성 증가
```

#### 전략 1-3: 신규 판매자 온보딩 강화
```
목표:
신규 판매자가 처음부터 배송 품질 중요성 인식

실행:
1. 온보딩 프로세스 재설계
   기존: 계약 → 상품 등록 → 판매 시작
   개선: 계약 → 배송 교육 (필수) → 상품 등록

2. 배송 교육 콘텐츠
   - "왜 배송이 중요한가" (데이터 기반)
   - "어떻게 배송을 잘하는가" (실전 가이드)
   - "배송 문제 시 대처법"

3. 시뮬레이션
   - 첫 10건 주문 집중 모니터링
   - 문제 발생 시 즉시 개입

투자:
- 시간: 1개월
- 비용: 낮음

기대 효과:
- 신규 판매자 배송 품질 향상
- 조기 이탈 감소
- 플랫폼 전반 품질 향상
```

### 5.3 Phase 2: 판매자 등급제 & 인센티브

#### 전략 2-1: 판매자 등급제 도입
```
목표:
배송 품질을 플랫폼 평판에 반영

등급 체계:
Diamond (상위 10%):
- 평균 배송 7일 이내
- 지연율 3% 이하
- 리뷰 4.5+ (최근 100건)

Gold (상위 25%):
- 평균 배송 10일 이내
- 지연율 5% 이하
- 리뷰 4.0+

Silver (상위 50%):
- 평균 배송 12일 이내
- 지연율 8% 이하
- 리뷰 3.5+

Bronze (하위 50%):
- 그 외

혜택 (등급별):
Diamond:
- 검색 노출 우선순위 최상위
- 수수료 10% 할인
- 전용 계정 매니저
- "베스트 셀러" 배지

Gold:
- 검색 노출 우선순위 상위
- 수수료 5% 할인
- "인기 셀러" 배지

Silver:
- 기본 혜택

Bronze:
- 개선 지원 프로그램 제공
- 3개월 내 개선 안 되면 노출 제한

투자:
- 기술: 등급 시스템 개발
- 마케팅: 등급제 홍보
- 비용: 중간 (수수료 할인 포함)

기대 효과:
- 판매자 간 선의의 경쟁
- 전체 배송 품질 향상
- 구매자 신뢰 증가
```

#### 전략 2-2: 지역별 배송 지원 프로그램
```
목표:
물류 취약 지역 판매자 특별 지원

대상:
- 북동부, 북부 지역 판매자
- 배송 시간 전국 평균 대비 1.5배 이상

지원:
1. 배송비 보조 (일부)
   - Olist가 배송비 20% 부담
   - 단, 등급 Silver 이상만

2. 물류 파트너 연결
   - 지역 특화 택배사 제휴
   - 대량 계약으로 단가 인하

3. 풀필먼트 옵션
   - 대도시 창고 공동 활용
   - 재고 전진 배치

투자:
- 비용: 높음 (배송비 보조)
- 하지만 지역 확장 효과

기대 효과:
- 지역 격차 완화
- 새로운 시장 개척
- 판매자 만족도
```

### 5.4 Phase 3: 물류 인프라 투자 검토

```
- 막대한 초기 투자

장점:
- 배송 속도 획기적 개선

단점:
- 건설/운영 비용

Option B-3: 물류 파트너십

권장:

시장 반응 보고 Phase 3 결정

단, 지속적 모니터링:
- 경쟁사 물류 투자 동향
- 판매자 배송 품질 개선 속도
- 구매자 만족도 변화
- 자금 조달 가능성
```

```
△ 데이터 분석은 잘함
△ 하지만 비즈니스 연결 약함
△ 또는 비즈니스는 이해하는데 데이터 활용 약함

→ "잠재력은 있지만 한 영역만 강함, 보완 필요"
```

#### Fail (불합격)
```
❌ 데이터 기본 해석 틀림
❌ 비즈니스 모델 이해 없음
❌ 실행 불가능한 제안
❌ 한계 인식 없음 (과신)
❌ 질문에 답변 못 함

→ "현재 역량으로는 불가"
```
